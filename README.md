## About

**About Me** is a demo app that displays personal information:

- **Name** — loaded from the `MyName` data class
- **Nickname** — entered by the user via `EditText`, stored through two-way DataBinding (`@={}`)
- **Profile image** — displayed with `ImageView`
- **Biography** — long text inside a `ScrollView`

When the user taps **Done**, the nickname is saved to the data class, the `EditText` and button are hidden, and the nickname `TextView` becomes visible.

---

## Architecture

**Pattern:** Single Activity + DataBinding

```
MainActivity
    └── binding: ActivityMainBinding
            └── myName: MyName
                    ├── name: String
                    └── nickname: String
```

| Component | Role |
|---|---|
| `MyName` | Data class — holds `name` and `nickname` |
| `ActivityMainBinding` | Generated by DataBinding from `activity_main.xml` |
| `DataBindingUtil.setContentView()` | Inflates layout and creates binding object |
| `@={}` two-way binding | Keeps `EditText` text in sync with `MyName.nickname` |
| `invalidateAll()` | Forces the binding to re-read data and refresh the UI |

## Key Concepts

| Concept | How it is used |
|---|---|
| `LinearLayout` | Root layout — stacks all views vertically |
| `ScrollView` | Wraps the biography `TextView` to make it scrollable |
| `DataBinding` | Eliminates `findViewById`; binds data directly in XML |
| Two-way binding `@={}` | Writes user input from `EditText` back to `MyName.nickname` |
| `View.GONE` / `View.VISIBLE` | Toggles `EditText` + Button vs nickname `TextView` |
| `InputMethodManager` | Hides the soft keyboard when Done is tapped |
